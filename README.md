# SWEN Final Project

## Project Title

EchoEats – Dynamic Pricing Food Waste Marketplace

## Project Overview

EchoEats is a marketplace designed to reduce food waste by allowing restaurants, cafes, and bakeries to sell surplus food at discounted prices before it expires. As the expiration time approaches, the price of the food automatically decreases, encouraging consumers to purchase it instead of letting it go to waste.

This platform benefits both vendors and consumers. Vendors reduce food waste and recover some revenue, while customers can buy affordable meals.

# User Needs

## Consumers

Consumers want an easy way to find affordable food deals while still having clear information about what they are buying. Users should be able to discover nearby offers and quickly decide if the item is worth purchasing.

Consumers need to see important details such as the food description, current price, and expiration time so they know how fresh the item is. Since prices change as expiration approaches, consumers should receive notifications when prices drop or when items are close to their expiration date. Finally, the payment process should be simple and secure so users can complete purchases easily through the application.

## Vendors

Vendors such as restaurants, cafes, and bakeries need a simple way to sell surplus food that would otherwise be wasted. They should be able to list items quickly with minimal effort and allow the system to automatically adjust prices as the expiration time approaches.

This increases the chances that food will be sold instead of thrown away. Vendors need the ability to receive payments securely after a sale is completed as well.

# User Stories

## Consumer User Stories

* As a consumer, I want to find affordable food deals near my location so I can save money.
* As a consumer, I want to see clear information about each food item including price, description, and expiration time.
* As a consumer, I want to receive notifications when prices drop or when items are close to expiring so I don’t miss good deals.
* As a consumer, I want to reserve food items before someone else purchases them.
* As a consumer, I want to pay easily and securely through the application.

## Vendor User Stories

* As a vendor, I want to list surplus food quickly so it can be sold instead of wasted.
* As a vendor, I want the system to automatically adjust prices as expiration approaches to increase the chances of selling the item.
* As a vendor, I want to track sales and see how much food waste I am reducing.
* As a vendor, I want to receive payments securely for sold items.
* As a vendor, I want to view analytics about sales and demand to better manage future listings.

# Functional Requirements

The system shall support the following functionalities:

* The system shall allow users to register as either consumers or vendors.
* Vendors shall be able to create food listings with photos, descriptions, base price, and expiration time.
* Consumers shall be able to search for nearby food deals using location-based search.
* The system shall automatically update prices over time as the expiration time approaches.
* Vendors shall be able to define a minimum acceptable price for their items.
* Consumers shall be able to reserve food items for a limited time period.
* The system shall prevent multiple users from reserving the same item at the same time.
* The system shall securely process payments between consumers and vendors.
* The system shall send notifications when prices drop or when items are close to expiration.
* The system shall collect data for vendor analytics and environmental impact tracking.

---

# Non-Functional Requirements

The system shall meet the following quality requirements:

* The system shall support near real-time updates for price changes.
* The application shall be scalable to support many users simultaneously.
* User data and payment information shall be securely stored and protected.
* The system shall maintain good performance even during peak usage hours.
* The interface shall be simple and easy to use on mobile devices.
* The system shall maintain logs for transactions and pricing history.

---

# Project Goal

The goal of EchoEats is to reduce food waste while providing affordable food options for consumers and helping vendors recover value from surplus food.

# EchoEats – Phase 2

## Description
EchoEats is a mobile-style web application that allows users to browse food listings, reserve items, and complete purchases.

## Features
- User registration and login
- Browse available food listings
- Add new listings with images
- Remove listings
- View item details
- Reserve items
- Checkout system
- Remove items from cart
- Dynamic item status (available → reserved → sold)

## Technologies Used
- HTML
- CSS
- JavaScript
- Local Storage
